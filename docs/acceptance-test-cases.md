## 第二部分：算法验证用例（5项）

### A01: 基本功能验证

| 项目 | 内容 |
|------|------|
| **参数** | 春季，12×12，5000G，选择：防风草 + 土豆 + 花椰菜 |
| **验证 H1（至少1株）** | 防风草 ≥ 1株，土豆 ≥ 1株，花椰菜 ≥ 1株 |
| **验证 H2（可达性）** | 从地图边缘BFS，所有作物格均可到达 |
| **验证 H3（喷水器覆盖）** | 每个作物格在至少1个喷水器的上下左右4格范围内 |
| **验证 H4（稻草人覆盖）** | 每个作物格在至少1个稻草人的13×13范围内 |
| **验证 G1（ROI）** | ROI > 0，且数值合理（详见下方预期分析） |
| **验证 G3（同类聚合）** | 同种作物大部分相邻种植（不要求100%聚合） |
| **预期ROI分析** | 三种作物单株ROI：花椰菜=2.19（最优）、防风草=1.75、土豆=1.60。算法按ROI降序贪心填充，但受网格容量限制（12×12=144格中约30%被喷水器/稻草人/通道占用，实际可种约80-100株作物）。预算5000G远超作物成本，因此实际种植数量受网格容量而非预算约束。综合ROI预计在1.5-2.5之间（接近花椰菜的2.19，因算法会优先多种花椰菜） |

## 第三部分：收益计算验证

### 计算公式

**不可重复收获作物**:
```
收获次数 = floor(27 / growthDays)
总产出 = baseSellPrice × 收获次数
总投入 = seedPrice × 收获次数
ROI = 总产出 / 总投入
```

**可重复收获作物**:
```
收获次数 = floor((28 - growthDays - 1) / regrowInterval) + 1
总产出 = baseSellPrice × 收获次数
总投入 = seedPrice（仅初始种子费）
ROI = 总产出 / 总投入
```

|------|------|
| **参数** | 春季，3×3，5000G，全选10种春季作物 |

| **预期** | HTTP 400, errorCode=GRID_TOO_SMALL（3×3网格无法放置10种作物+工具并满足H1-H4所有硬性约束） |
| **注意** | 如果只选1种作物，3×3可能可行（1作物+1喷水器+1稻草人=3格，稻草人13×13覆盖全图），因此必须多选作物才能触发此错误 |
